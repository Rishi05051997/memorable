Image

The Image component provides different states of Image responsiveness.

Image (Landscape-oriented)

Image in landscape mode means width of the image is more than height if we want to round that image then rounding of such image is somewhat difficult. So if user wants circular such kind of image then only apply circular--landscape class on parent div of img tag

Image (Portrait-oriented)

Image in Portrait mode means height of the image is more than width if we want to round that image then rounding of such image is somewhat difficult. So if user wants circular such kind of image then only apply circular--portrait on parent div of img tag

Image (Square)

User want to round a square image then only apply circular--square on parent div of img tag

Basic Responsive Image

We have a div that acts as the container of an img element HTML.The container has a width property of 96% so that it has left and right margins.It has a max-width of 960px so that the layout is not too wide on large screens. The element within the container has a width property of 100% so that its width is always equal to its container regardless of the viewport’s size, thus making it responsive. The height is set to auto so that the image proportionally scales. User only apply basic_img_responsive on parent div of img tag

Responsive Images in Columns

Sometimes we want images to display side-by-side in columns. For example, image galleries often display thumbnails on a grid. To achieve responsive images in columns, the only change we have to do is lower the CSS width property and give img elements a display property value of inline-block.

Two-column Responsive Image Layout

For a two-column responsive image layout, we can set the CSS width property to 48%, or roughly one-half of the container. The reason it’s not set to 50% is to give the images margins on their sides. HTML. User only apply basic_img_responsive_two_column on parent div of img tag.

Example

Two-column Responsive Image Layout

Three-column Responsive Image Layout

For a three-column responsive image layout, the concept is the same, we just need to set the width property to about one-third of the container’s width: 32%. HTML.User only apply basic_img_responsive_three_column on parent div of img tag.

Images with Conditional Breakpoints

It’s a good idea to have conditional breakpoints for responsive images that are displayed in columns so that when the images get too small, the columns will collapse. We’ll need media queries to make this happen. In the following example, the images will display in one column on smartphones, two columns on tablets, and four columns on larger screens. User only apply responsive_img_break_points on parent div of img tag.

Full Width Responsive Image

To have responsive images that are always 100% of the viewport’s size, we just need to remove the container’s max-width property (which is 960px) and give it a width of 100%.User only apply responsive_img_full_width on parent div of img tag.

Resolution Switching

In Case of resolution, there can be multiple resolutions that can be used. Based on differen resolutions, we can used different images. 100W 200W means actual or intrensic width of an image

srcset="https://picsum.photos/100 100W, https://picsum.photos/200 200W"

it is telling the browser that as we know every image has default size but now we put it insize a box or div then based on the container, we can change the size but by default we always inspect that then inspector always tell us correct size of an Image.

sizes="(min-width:600px) 200px, 600px"

Basically we are telling the browser before hand itself, when we have min-width of 600px, then the box which contains the image is of size 200px, otherwise by default size of an image is 600px
